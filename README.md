# Игра на реакцию для двух игроков на ATmega328P

Мой DIY-гаджет: игра на реакцию для двух игроков. После случайной задержки загорается центральный зелёный светодиод — кто быстрее нажмёт свою кнопку, тот выигрывает раунд. Первый до 3 очков выигрывает матч.

Проект на "голом" ATmega328P. Код написан в Arduino IDE (только core-функции), прошивка HEX-файлом через программатор T48



## Версии
- **v1.0** (стабильная, собрана и протестирована): простая индикация — один красный LED (P1) и один синий LED (P2) мигают при победе в раунде. При победе в матче — быстрое мигание индикатора победителя.
- **v2.0** (в разработке): улучшенная индикация — по три LED на игрока (красные для P1, синие для P2). Индикаторы счёта зажигаются последовательно и остаются гореть. При победе в матче — красивая поочерёдная анимация всех трёх индикаторов победителя.

## Механика игры (общая для обеих версий)
- Случайная задержка 1–5 секунд перед стартом.
- Анти-чит: если игрок нажмёт кнопку больше 2 раз во время ожидания — он блокируется на раунд.
- После загорания центрального зелёного LED — 5 секунд на реакцию.
- Победа в матче — до 3 очков.
- Стартовый сигнал: 3 моргания центрального LED.

## Схема и производство
- Полные проекты KiCad: папки `/hardware/v1` и `/hardware/v2`
- Gerber-файлы для заказа PCB: `/gerbers/v1_gerbers.zip` и `/gerbers/v2_gerbers.zip`

## Код и прошивка
Код написан в Arduino IDE, но использует только базовые функции (без дополнительных библиотек).

- Исходники: папки `/src/v1` (reaction_game_v1.ino) и `/src/v2` (reaction_game_v2.ino)
- Готовые HEX-файлы: `/firmware/v1.hex` и `/firmware/v2.hex`
